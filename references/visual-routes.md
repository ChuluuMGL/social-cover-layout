# Original Visual Routes

## adaptive-composite（默认唯一封面系统）

`adaptive-composite` 把“人物/标题优先”和“产品/证据优先”合并成一个有深度的组合系统。它不是两个模板，而是同一张封面里的主视觉、文字层和前景层协作。

### 内部策略

- `headline-first`：人物是点击锚点，标题是第一阅读对象，道具解释内容。适合教程、效率工具和个人经验。
- `proof-first`：产品、网站截图或电脑是主要证据，人物退到辅助层。适合工具发布、产品介绍和无人物场景。
- `bridge-hybrid`：标题采用 `headline-first` 的语义分组，主体采用 `proof-first` 的产品证据；适合 Website Skill、内容创作 Skill 等“人物 + 工具”主题。

### 自适应判断

- 标题短、钩子强、人物动作有信息：偏 `headline-first`。
- 标题包含产品名、界面/截图是主要证明：偏 `proof-first`。
- 同时有真人和产品证据：使用 `hybrid`，不牺牲标题可读性，也不丢掉产品证明。
- 不需要用户选择策略；Skill 先给出策略记录，再生成封面。

### 换行规则

- 4–8 个汉字或短英文产品名：优先一行。
- 9–14 个汉字：按语义边界拆成两行，不按字符宽度随机折断。
- 工具名 + 结果承诺：工具名可以独立成一组，结果承诺作为第二组；例如 `Website Skill / 快速建站`。
- 同一文字组中的汉字必须共享基线；人物或道具可以压住文字边缘，但不能让单个字下坠。

### 关键词层级规则

- 一张封面最多两组主视觉文字；每组最多拆成两行。
- 换行优先按语义拆分，例如 `Website Skill / 快速建站`、`写笔记 / 还能做封面`，不按字符宽度随机折断。
- 一组文字中只强调 1–2 个关键词：可以用第二种填充色、反白、底板或轻微字号差异，不要每个字都变色。
- 关键词变色和关键词换行可以同时使用，但必须保留一个明显的主阅读顺序。
- 英文产品名、数字或结果词可以成为独立视觉组；不要让工具名和结果承诺混成难以理解的一句话。

### 三层空间规则

所有封面先建立三层：

1. `background`：场景、渐变、网页氛围或纹理，只负责提供对比。
2. `type-and-proof`：标题、网页卡片、产品截图或证据主体。
3. `foreground`：人物、手、电脑前沿或一个道具，用于产生压字和深度。

三种层叠动作：

- `person-over-type`：人物肩膀、手或局部轮廓压住标题边缘；适合人物教程和经验分享。
- `type-over-object`：标题压住电脑、网页卡片或产品边缘；适合工具发布和产品证明。
- `split-layer`：标题的一组在主体后面，另一组在主体前面；适合长标题和需要强空间感的主题。

- 允许人物、电脑、网页卡片或产品截图遮盖标题边缘 5%–12%。
- 遮盖必须是清晰的前景/中景关系，不能用文字自身错位制造“动感”。
- 眼睛、嘴巴、关键手势和标题核心关键词不能被遮挡。
- 标题描边、硬阴影和轻微高光共同制造立体感；阴影方向保持一致，不要同时使用多套透视。
- 层叠不是把标题压到看不见：至少保留 88% 的主标题字形可读面积。

## 适配示例

### Website Skill

- 默认策略：`bridge-hybrid` + `split-layer`。
- 文字：`Website Skill` 作为第一组，`快速建站` 作为结果组；“建站”可以换第二种颜色。
- 视觉层：网页界面做中景证据，人物或电脑前沿做前景压字。

### 内容创作 Skill

- 默认策略：`bridge-hybrid` + `person-over-type`。
- 文字：`内容创作 Skill` 作为工具名，`写笔记还能做封面` 按语义拆成两行；“封面”可以使用强调色。
- 视觉层：人物承担亲和力，稿件卡片和封面卡片承担流程证明。

这两个案例使用同一套 `adaptive-composite`，差异来自内容证据和层叠动作，不来自安装两个 Skill。

## evidence-compare

用前后、两种方案、界面前后状态或一张关键截图证明内容。

- 对比标签短于主标题。
- 证据必须能在缩略图里看懂；不能用小字堆满画面。

## type-led

标题、数字或短语本身成为主体，背景退到第二层。适合排版教程、标题模板和方法论。

## editorial-photo

照片或场景负责情绪，标题只负责导航。适合品牌故事、生活方式和人物观点，不适合工具发布的强结果承诺。
