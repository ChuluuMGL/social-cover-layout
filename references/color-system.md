# Adaptive Color System

颜色是独立于排版策略的 Token。默认选择一个主标题色、一个辅助标题色和黑/深色描边，不把所有颜色同时堆进画面。

| 色彩方案 | 主色 | 辅助色 | 适合情绪 |
|---|---|---|---|
| `sunny-yellow` | `#FDFFA7` | 白色 | 教程、效率、轻快发现 |
| `signal-red` | `#FF5A5F` | 白色 | 警示、避坑、强观点 |
| `electric-blue` | `#60A5FA` | 白色 | 科技、产品、工具 |
| `fresh-green` | `#34D399` | 白色 | 方法、增长、清单、行动 |
| `warm-orange` | `#FF9F43` | 白色 | 推荐、生活方式、轻松经验 |
| `neon-pink` | `#F472B6` | 白色 | 创意、设计、年轻化内容 |
| `soft-lavender` | `#C4B5FD` | 深紫色 | 设计、编辑器、创作工具；更柔和 |
| `mint-sky` | `#A7F3D0` | `#60A5FA` | 方法、工作流、产品上手；清爽舒适 |
| `coral-cream` | `#FB7185` | `#FED7AA` | 个人表达、生活方式、轻内容；有温度 |
| `deep-blue-lime` | `#93C5FD` | `#BEF264` | 科技、建站、工具升级；沉稳中有抓眼点 |

## 选择规则

- 深色背景优先使用浅色填充 + 黑色描边；浅色背景优先使用深色填充或深色底板。
- 默认只用一种高饱和强调色；需要双色时，第二色只用于关键词或第二行标题。
- 亮色方案用于信息流停留，柔和方案用于舒适阅读；两者都必须通过深色描边、深色底板或明度差保持缩略图可读。
- 双色方案的第二颜色只用于一个关键词、一个结果词或一个前景道具，不把整张图变成彩虹。
- 品牌色优先于自动配色，但必须通过明度/描边保证手机缩略图可读。
- 人物、产品和背景已经很丰富时，标题不要再使用三种以上填充色。
- 色彩不能改变标题语义；`快速建站` 仍然要表达建站，不要让颜色制造“图片生成”误读。
